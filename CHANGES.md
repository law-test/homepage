## ✅ index.html에서 적용한 수정

### 1. 로고 반영
- 헤더 좌측에 로고 이미지 삽입 (`assets/images/logo.png`)
- 푸터에도 로고 사용 (반전 처리해 검정 배경에서 표시)
- 로고의 골드(#B5945C) + 블랙(#0F0E0C) 색상을 사이트 전체 컬러 토큰으로 채택

### 2. 시점 불일치 (대회 종료 반영)
- 상단 다크바: `참가신청` → `제7회 시행 완료 · 결과 발표 예정` 상태 표시로 교체
- 새 알림 배너 추가 ("제7회 응시자 여러분께 감사드립니다…")
- 히어로 메인 CTA: `참가 신청` → `결과 발표 안내` (notice.html 연결)
- 보조 텍스트로 "제8회 사전 알림 신청은 추후 공지" 명시
- 대회 일정 섹션을 시각적 타임라인으로 재구성 (Ⅰ·Ⅱ는 ✓ 완료, Ⅲ은 진행 중, Ⅳ는 대기)

### 3. 공지사항 정렬 오류 수정
- 기존: 04.20 → 04.15 → 04.10 → 05.01 → 03.20 (뒤죽박죽)
- 수정: 05.01 → 04.20 → 04.15 → 04.10 → 03.20 (최신순)

### 4. 로마숫자 섹션 번호 일관성
- Three Pillars: Ⅰ · Ⅱ · Ⅲ 유지
- Ceremony Archive: 기존 `Ⅵ` → `Ⅳ`로 수정 (연속된 섹션 번호)
- "6회의 기록"은 부제 `Six Ceremonies`로 분리해서 표시

### 5. 중복 정보 제거
- 기존: 히어로 + 포스터 섹션에 Date/Format/Volume 정보가 두 번 반복
- 수정: 히어로의 4-칸 메타 그리드(`Date · Format · Subjects · Volume`)에 한 번만 표시. 포스터 섹션은 설명문만 남김

### 6. 주소 오타 수정
- `제지하1층 29호` → `지하1층 29호`

### 7. 첫 화면에 핵심 정보 노출
- 4-칸 메타 그리드로 일정·방식·과목·분량을 즉시 확인 가능
- 시상식 아카이브 카드에 회차별 점수 명시 유지

### 8. 외부 링크에 대한 신뢰성 보강
- "제8회 사전 알림 신청은 추후 공지됩니다" 안내 문구 추가
- 외부 결제 링크(스마트스토어)로의 즉각 이동을 줄이고 자체 페이지(notice.html, contact.html) 우선

### 9. 라틴 숫자 + 아라비아 숫자 병기
- `VOL. VII · 제7회 · 2026` 형태로 핵심 숫자 정보 병기
- 장식적 로마숫자(MMXXIV, MMXXVI, Ⅰ~Ⅵ)는 유지하여 편집 디자인 톤 보존

### 10. 인용문 시점 명시
- 김승현 부회장 발언에 `제3회 시상식 축사 · 2025.05.10` 컨텍스트 추가

### 11. 추가 개선
- **타이포그래피**: Noto Serif KR + Pretendard + Cormorant Garamond 조합으로 격조 있는 편집 디자인 톤
- **반응형**: 모바일/태블릿 레이아웃 대응 (햄버거 메뉴 포함)
- **접근성**: skip link, semantic HTML, alt 텍스트, `prefers-reduced-motion` 지원
- **성능**: 이미지 `loading="lazy"`, 폰트 `preconnect`
- **인터랙션**: 부드러운 페이드업 애니메이션, hover 마이크로 인터랙션, sticky 헤더

## 🔧 다른 페이지 (about.html, achievements.html)에서 직접 고치셔야 할 곳

### about.html

**A. 목차에 없는 Ⅵ 섹션 + Ⅴ와 내용 중복**
- 본문의 `Ⅵ. 순환 학습 플랫폼 (A Self-Reinforcing Cycle)` 섹션은 Ⅴ와 같은 6단계 사이클을 다룹니다.
- **권장**: Ⅵ 블록을 Ⅴ 섹션 내부의 다이어그램으로 통합. 또는 별도 섹션으로 두려면 목차에도 추가하고 제목을 차별화 (예: `Ⅴ. 순환식 교육 구조의 의미` / `Ⅵ. 6단계 사이클 다이어그램`).

**B. 회차별 점수 통계 갱신**
- 현재: `회차별 평균 33~44점, 최고점 70~73점` (1~6회 기준)
- **권장**: 7회 결과 반영하거나 `(제1~6회 기준)` 단서 명시

**C. 회차 수치 변수화**
- "출범 이후 7회에 걸쳐 시행" 같은 본문은 매 회차마다 갱신해야 합니다.
- **권장**: Jekyll의 `_config.yml`에 `latest_round: 7` 같은 변수 추가 → 본문에서 `{{ site.latest_round }}회`로 참조

### achievements.html

**A. 박○○ vs 참여 학습자 A 중복 (가장 시급)**
- "전국 자기주도 학습법 공모전" 섹션에서 같은 인물의 성과가 위·아래 두 곳에 다른 라벨로 나열됩니다.
  - `대상 박○○: LEET 30점 상승, 우수학술지 논문 게재`
  - `참여 학습자 A: LEET 30점 상승, 토익 200점 상승, 우수 학술지 논문 게재`
  - `최우수상 한정현: 전일제 직장 + 주경야독, 변호사시험 기출해설서 포함 저서 6권`
  - `참여 학습자 B: 전일제 직장 재직 중 변호사시험 해설서 등 저서 6권 집필`
- **권장**: 하단의 `참여 학습자 A`, `참여 학습자 B` 블록을 통째로 삭제

**B. 익명/실명 처리 비일관성**
- 대상은 `박○○`인데 최우수상 3명(한정현·이승관·김준이)은 실명입니다.
- **권장**: 본인 동의 기준으로 통일. 가장 비중 있는 수상자만 익명인 것은 어색합니다.

**C. 합격증 사진 공개 — 개인정보 점검**
- `박상영 학생 서울대학교 대학원 법학과 석사과정 합격증` 사진이 그대로 노출됩니다.
- **권장 점검사항**:
  1. 본인의 명시적 공개 동의(서면)가 있는지
  2. 수험번호·생년월일 등 식별정보 마스킹 처리했는지
  3. 가능하면 합격증 대신 본인 인용문 + 사진으로 대체

**D. 논문 출판 시점 표기**
- `대법원 2026.3.12 선고 2025도12709 판결 ... 논문 게재` — 2개월 만에 등재지 게재는 매우 빠른 일정입니다.
- **권장**: 게재 확정이 아니라 투고/심사 중이라면 표기 구분. KCI 링크 동작 여부도 확인.

## 🛠 권장 리팩토링 (Jekyll 기반이라면)

### `_includes/footer.html` 분리
모든 페이지 푸터가 동일하게 반복됩니다. 다음과 같이 분리하세요:

**`_includes/footer.html`**
```html
<footer class="site-footer">
  <!-- (footer 내용) -->
</footer>
```

**각 페이지**
```liquid
{% include footer.html %}
```

→ "지하1층" 같은 수정이 한 번에 모든 페이지에 반영됩니다.

### `_config.yml`에 회차 변수
```yaml
# _config.yml
latest_round: 7
latest_round_roman: "Ⅶ"
contest_year: 2026
contest_date: "2026.5.9"
```

페이지에서 `{{ site.latest_round }}회`처럼 참조하면 다음 회차 갱신 시 한 줄만 바꾸면 됩니다.

### `_data/notices.yml`로 공지 관리
```yaml
# _data/notices.yml
- date: 2026-05-01
  type: press
  title: 제7회 법학경시대회 개최 보도자료
  url: media.html
- date: 2026-04-20
  type: important
  title: 제7회 법학경시대회 시행계획 공고
  url: notice.html#notice-7
# ...
```

자동 정렬 + 추가/삭제가 쉬워집니다.

```css
  --paper-warm: #F4EEDF; /* 카드 배경 */
  --hairline: #E0D7C4;   /* 가는 선 */
}
```

- 한국어 디스플레이: Noto Serif KR
- 한국어 본문: Pretendard
